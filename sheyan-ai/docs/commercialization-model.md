## 1. 这份文档解决什么问题

你要在商业化上搞清楚五件事：

| 问题 | 本文档给出 |
|------|------------|
| 什么时候能回款？ | 分阶段回本时间表 + 付费人数门槛 |
| 毛利多少？ | 按套餐/API 成本的毛利率公式与测算 |
| 纯利多少？ | 扣除固定成本、获客、支付手续费后的净利 |
| 比率是多少？ | API 成本率、毛利率、净利率、LTV/CAC 等 |
| 未来怎么加价？ | 增值服务矩阵与定价逻辑 |

**重要说明：** 下文「API 成本」来自 2026-06 实测数据（Neon 数据库 19 次成功调用 + QuickRouter 账户历史消耗 ¥1.86 对账）。售价、付费率、固定成本为**可调整的假设**，你改假设就能重算整张表。

### 2.3 成本波动（为什么 flat 定价有风险）

同扣 1 点的文本对话，API 成本可差 **70 倍**：

| 分位 | 总 Token | 估算 API 成本 |
|------|----------|---------------|
| 最短 | 444 | ≈ ¥0.003 |
| 中位 P50 | 8,189 | ≈ ¥0.055 |
| P90 | 29,326 | ≈ ¥0.20 |
| 最长 | 31,613 | ≈ ¥0.21 |

v1.1.0 工作区（项目记忆、场景模板、长上下文）显著抬高输入 Token，是成本上升主因。

### 2.4 QuickRouter 充值折扣（已纳入成本模型）

QuickRouter 按 **¥1 = ¥1 额度** 面值计费，充值时有批量折扣（你截图中的「9.90 折」等）：

| 充值金额 | 折扣 | 实付系数 | 有效 API 成本变化 |
|----------|------|----------|-------------------|
| ¥100 | 9.90 折 | **0.99** | 名义成本 × 0.99 |
| ¥500 | 9.85 折 | 0.985 | × 0.985 |
| ¥1,000 | 9.80 折 | 0.98 | × 0.98 |
| ¥10,000 | 9.60 折 | 0.96 | × 0.96 |
| ¥100,000 | 9.45 折 | 0.945 | × 0.945 |

**平台已配置项（系统开关）：**

- `quickrouter_recharge_discount`：默认 **0.99**（按你当前 ¥100 档充值）
- `api_cost_cny_per_1k_tokens`：默认 **0.0067**
- `api_cost_image_cny`：默认 **0.41**

每次对话成功后会写入 `UsageRecord.estimatedCost`（已含折扣系数），后台 **用量分析** 页可查看。

**定价启示：**

- 你充值越多，真实 API 成本越低 → **毛利越高**
- 卖价不应按「无折扣名义成本」算，应按 **你实际充值档位的有效成本** 算
- 例：名义 ¥0.081/点 × 0.99 ≈ **¥0.080/点** 有效成本（9.9 折档）
- 若升级到 ¥10,000 档（9.6 折），有效成本再降 **4%**

## 3. 核心公式（背下来就能自己算）

### 3.1 单行利润

```
用户支付（收入）= 售出点数 × 每点售价
API 变动成本     = 消耗点数 × 每点 API 成本（实测 ≈ ¥0.081）
毛利             = 收入 − API 变动成本
毛利率           = 毛利 ÷ 收入 × 100%
```

### 3.2 纯利（经营层面）

```
纯利 = 毛利 − 固定成本 − 获客成本 − 支付手续费 − 其他运营支出
净利率 = 纯利 ÷ 收入 × 100%
```

### 3.3 关键比率

| 比率 | 公式 | 健康参考 |
|------|------|----------|
| **API 成本率** | API 支出 ÷ 充值收入 | < 60%（你目标 < 55%） |
| **毛利率** | (收入 − API 成本) ÷ 收入 | ≥ 40%（SaaS 工具类） |
| **净利率** | 纯利 ÷ 收入 | 内测期 ≥ 15%，稳定期 ≥ 25% |
| **赠送成本率** | 注册赠送 API 成本 ÷ 当月收入 | 早期可高，规模化后 < 10% |
| **付费转化率** | 付费用户 ÷ 注册用户 | 内测 5–15%，成熟 10–20% |
| **ARPU** | 月收入 ÷ 付费用户数 | 随套餐而定 |
| **LTV** | ARPU × 平均留存月数 × 毛利率 | 应 > CAC × 3 |

## 4. 定价与套餐设计（建议版）

### 4.1 成本锚点

```
API 成本锚点：¥0.08 / 点（保守按 ¥0.081 取整）
```

### 4.2 推荐零售价（人工充值阶段）

| 套餐 | 点数 | 售价 | 每点售价 | 预估 API 成本 | **毛利率** |
|------|------|------|----------|---------------|------------|
| 体验包 | 50 | ¥9.9 | ¥0.198 | ¥4.0 | **59.6%** |
| 标准包 ⭐ | 300 | ¥49 | ¥0.163 | ¥24.3 | **50.4%** |
| 专业包 | 800 | ¥99 | ¥0.124 | ¥64.8 | **34.5%** |
| 工作室包 | 2000 | ¥199 | ¥0.100 | ¥162 | **18.6%** ⚠️ 走量档，毛利偏薄 |

> ⭐ **主力套餐建议：¥49 / 300 点**，用户感知「1 次对话约 ¥0.16」，你方毛利约 **50%**。

### 4.3 注册赠送建议（控获客成本）

| 赠送点数 | 预估 API 成本（全用于文本） | 说明 |
|----------|----------------------------|------|
| 20 点（现状） | ≈ ¥1.62 | 偏大方，长对话用户可能到 ¥2–4 |
| **15 点（建议）** | **≈ ¥1.22** | 够 10+ 轮体验，成本可控 |
| 10 点（保守） | ≈ ¥0.81 | 适合严格控 CAC 阶段 |

**获客成本（CAC）估算（仅 API 赠送）：**

```
CAC_api ≈ 赠送点数 × ¥0.081
15 点 ≈ ¥1.22 / 注册用户
```

若加上人工运营、内容、你的时间，真实 CAC 建议按 **¥3–8 / 注册** 做预算（内测期）。

## 6. 什么时候能回款？（回本测算）

### 6.1 先定义「回款」

| 层级 | 含义 | 你怎么判断 |
|------|------|------------|
| **API 现金流回本** | 当月充值收入 > 当月 API 支出 | 后台看 QuickRouter 消耗 vs 人工收款 |
| **运营成本回本** | 月收入 > API + 固定成本 + 支付费 | 表格 Section 6.3 |
| **全成本回本** | 上述 + 人力 + 获客摊销 | 有稳定兼职收入后可算 |

内测期优先盯：**API 现金流回本** → **运营成本回本**。

### 6.2 场景假设

| 参数 | 保守 | 基准 ⭐ | 乐观 |
|------|------|---------|------|
| 月新增注册 | 10 | 30 | 80 |
| 付费转化率 | 5% | 10% | 15% |
| 客单价（首充） | ¥9.9 | ¥49 | ¥49 |
| 月复购率 | 10% | 20% | 30% |
| 注册赠送 | 15 点 | 15 点 | 15 点 |
| 固定成本/月 | ¥50 | ¥100 | ¥200 |

### 6.3 月度损益测算（基准场景）

**假设：每月 30 注册，10% 付费 → 3 人首充 ¥49，20% 老用户复购 1 人**

| 项目 | 计算 | 金额 |
|------|------|------|
| 首充收入 | 3 × ¥49 | ¥147 |
| 复购收入 | 1 × ¥49 | ¥49 |
| **月总收入** | | **¥196** |
| 售出点数 | 4×300 | 1,200 点 |
| API 成本（用户消耗） | 1,200 × ¥0.081 | ¥97.2 |
| 注册赠送 API | 30 × 15 × ¥0.081 | ¥36.5 |
| **API 总支出** | | **¥133.7** |
| **毛利** | 196 − 133.7 | **¥62.3** |
| **毛利率** | | **31.8%** ⚠️ 被赠送拉低 |
| 固定成本 | | −¥100 |
| **月纯利（经营）** | | **−¥37.7** |

> 基准场景下 **月收 ¥196 仍略亏**，主要因为：付费人数少 + 赠送 30 人吃掉 ¥36.5。

**基准场景要月度打平（纯利 ≥ 0）需要：**

```
月收入 ≥ API 消耗 + 赠送成本 + 固定成本
设每月付费 4 人首充 ¥49，无复购：
收入 ¥196，售出 1200 点
用户消耗 API ≈ ¥97，赠送 30 人 ≈ ¥36.5，固定 ¥100
→ 196 − 97 − 36.5 − 100 = −¥37.5（仍亏）

需要约 6–7 个 ¥49 付费/月（≈1800–2100 点售出），且注册控制在 30 以内：
收入 ≈ ¥294–343
API 用户消耗 ≈ ¥146–170
赠送 ≈ ¥36.5
固定 ≈ ¥100
纯利 ≈ ¥294 − 146 − 36.5 − 100 ≈ ¥11.5（刚打平）
```

### 6.4 回款时间表（可执行预期）

| 阶段 | 时间 | 目标 | 收入门槛（月） | 你能回款什么 |
|------|------|------|----------------|--------------|
| **Phase 0 内测** | 现在 | 10 个真实用户、稳定对话 | ¥0–200 | 先回 **API 预充值**（少亏） |
| **Phase 1 首单验证** | 1–2 个月 | 5 人愿意付 ¥49 | **¥200–300/月** | API 成本与收入大致持平 |
| **Phase 2 小规模盈利** | 3–4 个月 | 10+ 付费，复购出现 | **¥500–800/月** | 覆盖固定成本，**月纯利 ¥100–300** |
| **Phase 3 副业收入** | 6 个月 | 50+ 付费累计 | **¥2,000+/月** | **月纯利 ¥500–1,000** |
| **Phase 4 可持续** | 12 个月 | 在线支付 + 会员 | **¥5,000+/月** | 净利率 20%+，可考虑专职 |

**一句话：**

> 按 ¥49/300 点、API ¥0.08/点测算，**月收入 ~¥300** 可打平运营成本；**月收入 ~¥500** 开始有可观纯利；**月收入 ¥2,000** 才算「副业级」回款。

### 6.5 累计回本（你已投入的 QuickRouter 等）

| 已投入（示例） | 金额 |
|----------------|------|
| QuickRouter 历史消耗 | ¥1.86 |
| 当前余额（资产） | ¥8.34 |
| 开发时间（自研） | 不计入现金流则忽略 |

若只算现金：**再收 ¥10–50 首充**即可覆盖历史 API 试错成本（¥1.86 已花掉）。

## 8. 未来增值服务矩阵

### 8.1 分层逻辑

```
通用能力便宜（引流） → 垂直工作流贵（利润） → 团队/数据贵（LTV）
```

### 8.2 增值服务清单

| 层级 | 服务 | 用户痛点 | 收费方式 | 预估毛利 | 优先级 |
|------|------|----------|----------|----------|--------|
| **L1 点数增值** | 点数加量包 / 自动续点 | 不够用 | ¥9.9–99 一次性 | 35–60% | P0 |
| **L1** | 生图高清 / 多尺寸 | 出图质量 | +2–3 点/张 | 50%+ | P1 |
| **L2 会员订阅** | 月卡：每月 200 点 + 工作区 | 可预期支出 | ¥29–39/月 | 55%+ | P2 |
| **L2** | 年卡：折扣 + 专属模板 | 长期用户 | ¥299/年 | 60%+ | P2 |
| **L3 垂直工作流** | PRD 生成器（结构化导出） | 设研刚需 | 5–10 点/份 或会员含 | 70%+ | P1 |
| **L3** | 竞品截图分析 | 设计调研 | 8 点/次 | 65%+ | P1 |
| **L3** | 设计规范检查 | 交付质检 | 10 点/次 | 70%+ | P2 |
| **L4 内容资产** | 场景模板市场 | 省 prompt 时间 | ¥1.9–9.9/套 | 90%+ | P2 |
| **L4** | 团队共享项目记忆 | 协作 | ¥99/团队/月 | 80%+ | P3 |
| **L5 企业** | 多席位 + 统一账单 + 报表 | 工作室 | ¥499+/月 | 70%+ | P3 |
| **L5** | API 开放平台 | 集成自有系统 | 按 Token 加价 30% | 30% | P3 |
| **L5** | 私有化部署 | 数据敏感客户 | 项目制 ¥5万+ | 50%+ | P4 |

### 8.3 会员 vs 点数（怎么选）

| | 点数制 | 会员制 |
|--|--------|--------|
| 适合 | 低频、试用、不确定用量 | 高频、设研日常用户 |
| 现金流 | 预收好 | 稳定 MRR |
| 用户感知 | 「用多少买多少」 | 「包月划算」 |
| 建议 | **现在主力** | 月活 200+ 再上 |

**推荐组合：点数为主 + 月卡为辅**（月卡 200 点 ≈ 零售价 ¥32 的折扣版，标价 ¥39 制造锚点）。

## 10. 风险与敏感因素

| 风险 | 影响 | 应对 |
|------|------|------|
| gpt-5.5 涨价 | API 成本率上升 | 换模型档、分模式扣点、售价联动 |
| 长上下文滥用 | 单点成本 ¥0.2+ | 工作区模式 2–5 点、历史条数上限 |
| 只赠送不付费 | CAC 白烧 | 降赠送至 10–15 点、邀请制 |
| ¥99/800 大包卖太多 | 毛利跌破 35% | 限量或调高门槛 |
| 人工充值瓶颈 | 收入天花板 | 优先接微信收款码 + 自动到账 |
| session/API Key 泄露 | 成本失控 | 密钥轮换、额度告警 |

# 设研AI · 用增与商业化路线图

> 记录时间：2026-06  
> 适用阶段：内测 / 小范围试运营

---

## 平台现状摘要

| 维度 | 现状 |
|------|------|
| 认证 | 邮箱 + 密码注册（无邮箱验证） |
| 计费 | 平台点数制，文本/生图按次扣点 |
| 充值 | **人工充值**（用户报邮箱，管理员加点） |
| API | QuickRouter（gpt-5.5 文本 + gpt-image-2 生图） |
| 成本 | 平台点数 ≠ API 额度，需分别管理 |

---

## 优先级总览

```
稳定性 > 注册体验 > 人工充值标准化 > 场景引导 > 邀请裂变 > 在线支付 > 会员订阅 > 企业版
```

---

## P0 — 跑通闭环（现在 ~ 1 个月）

**目标：有人用、用得动、亏得起**

### 1. 核心体验稳定
- [x] 对话 / 生图 / 多模态输入
- [x] 切换对话不闪屏
- [x] QuickRouter API 错误提示准确
- [ ] 持续监控 API 成本 vs 扣点定价

### 2. 新用户注册送点
- [x] 默认赠送 20 点（后台可配 `register_welcome_points`）
- [x] 流水类型 `system_gift`
- [x] 注册成功绿色提示

### 3. 人工充值流程标准化
- [ ] 充值弹窗写清：套餐、价格、到账时间、联系渠道（微信/邮箱）
- [ ] 后台一键加点 + 备注（已有余额流水页，补 SOP 即可）
- [ ] 用户报邮箱 → 管理员 5 分钟内到账（设预期）

### 4. 单位经济算清

| 项目 | 建议核算 |
|------|----------|
| 文本 1 点 | ≈ QuickRouter 单次对话成本 |
| 生图 5 点 | ≈ gpt-image-2 单次成本 |
| 注册送 20 点 | 控制在 ¥1~3 API 成本以内 |
| 毛利率 | 售价点数 × 单价 − API 成本 ≥ 40% |

---

## P1 — 验证留存（50~200 真实用户）

**目标：让人回来、愿意付费**

### 5. 场景化引导（设研垂直场景）
首页 / 新对话示例 prompt：
- 「帮我写一份 XX 功能的 PRD」
- 「分析这张竞品截图的交互逻辑」
- 「生成一张科技感产品概念图」

### 6. 用量可视化
- 用户侧：今日已用 X 点，预计还能聊 Y 次 / 生 Z 张图
- 余额 < 10 点时温和提醒（非弹窗轰炸）

### 7. 邀请机制（轻量裂变）
- 邀请码 / 推荐链接：邀请人 + 被邀请人各得 N 点
- 适合设计圈、产品圈口口相传，成本低

### 8. 定价套餐草案（先文字，不接支付）

| 套餐 | 点数 | 定位 |
|------|------|------|
| 体验包 | 50 点 | 低价引流 |
| 标准包 | 200 点 | 主力套餐 |
| 专业包 | 500 点 | 高频用户 |

---

## P2 — 商业化成型（月活 500+ 或想月收入破万）

**目标：少靠人工、多靠系统**

### 9. 在线支付
- 微信 / 支付宝（人工充值变补充渠道）
- 支付成功自动到账点数

### 10. 会员订阅（可选）

| 类型 | 权益 |
|------|------|
| 月卡 | 每月固定点数 + 生图优先 |
| 年卡 | 折扣 + 专属模板 / 场景 |

### 11. 企业 / 团队版
- 多人账号、统一充值、用量报表
- 目标客群：设计工作室、小产品团队

### 12. 垂直功能溢价
- PRD 模板库、设计规范检查、竞品分析框架
- **原则：通用聊天便宜，垂直工作流贵**

---

## P3 — 规模化（PMF 明确后）

- [ ] 邮箱验证 + 忘记密码
- [ ] 微信登录（国内 C 端）
- [ ] API 开放平台（B 端）
- [ ] 内容营销：小红书 / 公众号案例

---

## 现在最该做的三件事

1. **注册送点 + 充值说明** — 让用户 3 分钟内体验到价值  
2. **定简单定价** — 如 ¥9.9 = 100 点，先人工收款  
3. **找 10 个真实用户内测** — 设计 / 产品背景，看是否愿意续费

| 优先级 | 功能 | 预估工作量 | 价值 |
|--------|------|-----------|------|

| P1 | 场景示例 / 模板入口 | 2 天 | 高 |
| P1 | 用量统计展示（用户侧） | 1 天 | 中 |

## 关键指标（建议每周看）

| 指标 | 说明 |
|------|------|
| 注册数 | 新用户增长 |
| 激活率 | 注册后 24h 内发过消息的比例 |
| 次日留存 | 第二天还回来的比例 |
| 付费转化率 | 充值用户 / 注册用户 |
| ARPU | 人均收入 |
| API 成本率 | API 支出 / 充值收入（应 < 60%） |
| 赠送消耗 | 注册送点带来的 API 成本 |

## 备注

- 邮箱注册已有，**邮箱验证现阶段不急**（没点数用不了，后台可禁用账号）
- QuickRouter API 额度与平台点数分开管理，需在后台监控 API 余额
- 内测期可用 `register_enabled` 关闭公开注册，改邀请制

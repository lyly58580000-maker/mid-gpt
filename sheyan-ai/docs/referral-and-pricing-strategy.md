## 2. 老带新（一级邀请）

### 2.1 为什么现阶段值得做

- 设计 / 产品圈适合口口相传，成本低
- 15 点注册赠送 API 成本约 **¥1.2/人**
- 若双方各 10 点邀请奖励 ≈ **¥1.6/有效新用户**
- 只要 **10% 付费率、客单价 ¥49**，单付费用户毛利约 **¥25**，可覆盖 **15+** 个邀请来的免费用户

### 2.2 推荐最小方案（P1，约 1–2 天开发）

```
用户 A 有邀请码 / 邀请链接（?ref=CODE）
→ 用户 B 注册时填写或 URL 自动带入
→ B 注册成功：获得注册点数（建议 15 点，或与裸注册区分）
→ A 获得：+10 点邀请奖励
→ 后台：邀请关系表、邀请人数、发出点数统计
```

### 2.3 防刷底线

- 被邀请人 **首次成功对话后** 再给邀请人发奖（或双方都在「有效激活」后发）
- 同 IP / 同设备注册频率限制
- 邀请人每月奖励上限（如 **50 点/月**）
- 禁止自邀、批量小号（简单规则 + 人工抽查）

### 2.4 暂不做

| 能力 | 原因 |
|------|------|
| 二级代理 / 下家拉人返佣 | 规则复杂、易薅羊毛、无人审 |
| 商家渠道积分体系 | 需月活 500+、专人运营 |
| 现金提现 / 分销结算 | 合规与财务成本高 |

### 2.5 何时升级二级 / 渠道

同时满足再考虑：

1. 月活 **200+**，付费用户 **30+**
2. 有专人处理作弊与纠纷
3. 明确渠道伙伴（工作室、培训机构）且愿意签简单协议

## 3. 锚定定价（高开低走）

### 3.1 成本锚点（2026-06 实测）

| 项目 | 数值 |
|------|------|
| 有效 API 成本 | ≈ ¥0.08/点（含 QuickRouter 9.9 折充值） |
| 健康零售价 | ¥0.15–0.20/点 |
| 主力套餐目标毛利 | **40–50%** |

### 3.2 推荐标价结构

| 档位 | 标价（锚点） | 活动 / 内测价 | 点数 | 备注 |
|------|--------------|---------------|------|------|
| 体验包 | ~~¥19.9~~ | **¥9.9** | 50 | 仅拉新，勿常态 |
| 标准包 ⭐ | ~~¥69~~ | **¥49** | 300 | 主力售卖价 |
| 专业包 | ~~¥149~~ | **¥99** | 800 | 走量档，毛利约 35% |

日常展示「划线价 + 内测伙伴价」，用户感知优惠，你仍按目标毛利售卖。

### 3.3 合规注意

- 划线价需有真实依据，避免虚构「原价 ¥999」类表述
- 推荐文案：**「标准价 ¥69，内测伙伴价 ¥49」**
- 活动价不宜常年天天有，否则用户只等打折

### 3.4 与邀请制组合示例

```
标价 ¥69 / 300 点
内测价 ¥49
+ 邀请码额外送 10 点
→ 用户感知超值，你仍收 ¥49，毛利健康
```

## 4. 开发 backlog（要做时勾选）

### 数据模型（建议）

- `User.inviteCode`：唯一邀请码（6–8 位）
- `User.invitedByUserId`：邀请人 ID（nullable）
- `InviteReward` 或扩展 `BalanceRecord`：`changeType = invite_reward`

### API

- `GET /api/user/invite` — 我的邀请码、链接、已邀请人数
- 注册 `POST /api/auth/register` 增加 `refCode` 参数
- `GET /api/admin/invites` — 邀请排行、发出奖励统计

### 配置项（系统开关）

- `invite_enabled`：是否开启邀请
- `invite_reward_inviter`：邀请人奖励点数（建议 10）
- `invite_reward_invitee`：被邀请人额外点数（建议 0 或 5，与注册赠送合并算）
- `invite_reward_on_first_chat`：是否首次对话后才发奖（建议 true）
- `invite_monthly_cap`：邀请人月奖励上限（建议 50）

### 前端

- 注册页邀请码输入 / URL `?ref=` 自动填充
- 用户中心「邀请好友」卡片 + 复制链接
- 管理后台邀请统计 Tab
